# 项目说明文档

本项目是一个 FastAPI + LangGraph 的车辆故障诊断服务。当前版本按用户要求简化为 6 个核心角色：

1. 车辆背景 Agent
2. 用户主观描述分析 Agent
3. DTC Agent
4. 知识库 Agent
5. 经验 Agent
6. 汇总 Agent

## 支持的 selected_analysts

```text
vin_context
symptom
dtc
knowledge
experience
```

默认启用全部 5 个前置分析 Agent，最后固定进入 `Summary Agent`。

## 当前图流程

```text
START
  -> VIN Context Analyst
  -> Symptom Analyst
  -> Diagnostic Code Analyst
  -> Knowledge Analyst
  -> Experience Analyst
  -> Summary Agent
  -> END
```

带工具的 Agent 会先请求工具，再回到对应 Agent 生成报告。

## 保留工具

| 工具 | 用途 |
| --- | --- |
| `get_vehicle_profile_by_vin` | 根据 VIN 查询车辆档案和里程 |
| `get_dtc_history_by_vin` | 根据 VIN 查询历史故障码 |
| `get_maintenance_history_by_vin` | 根据 VIN 查询维修/保养记录 |
| `lookup_dtc_code` | 查询单个 DTC 含义 |
| `search_dtc_combinations` | 查询故障码组合模式 |
| `retrieve_repair_cases` | 查询知识库/案例库 |

## 经验库

`Experience Analyst` 查询已经确认过的历史维修案例。写入动作不在诊断图里自动发生，而是在用户或业务系统提交确认维修结果后，通过：

```text
POST /api/vehicle-diagnosis/tasks/{task_id}/outcome
```

写入 memory，避免把未确认诊断污染经验库。

## 最终结果

最终由 `Summary Agent` 写入 `structured_result`。主要字段：

- `summary`
- `confidence_score`
- `ranked_hypotheses`
- `inspection_plan`
- `similar_experience_cases`
- `analyst_conclusions`
- `reports`
- `tool_errors`
- `diagnostic_trace`

## 已移除内容

当前版本已移除未在需求中提到的复杂 agent、传感器/事件工具，以及旧的多轮评审配置。
