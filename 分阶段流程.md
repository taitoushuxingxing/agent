# 简化分阶段流程

## 1. 接收任务

FastAPI 接收 VIN、用户症状、DTC、车辆信息和参数，创建诊断任务。

## 2. 顺序执行分析 Agent

```text
车辆背景 Agent
  -> 用户主观描述分析 Agent
  -> DTC Agent
  -> 知识库 Agent
  -> 经验 Agent
  -> 汇总 Agent
```

每个前置 Agent 只负责输出自己的证据和结论，不再进入辩论、维修建议或安全审查流程。

## 3. 工具调用

保留工具调用：

- VIN 车辆档案、历史故障码、维修保养记录
- DTC 字典和故障码组合查询
- 知识库/案例库查询
- 经验库查询

## 4. 汇总输出

汇总 Agent 根据前序报告生成：

- 最可能故障排序
- 支持证据和反证
- 下一步检查建议
- 置信度
- 完整 trace 和工具错误

## 5. 经验写入

维修完成并确认根因后，通过 outcome API 写入经验库。系统不会把未确认的推测自动写入经验库。
