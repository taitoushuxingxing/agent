# 车辆故障诊断 Agent 流程图

> Mermaid 流程图，可在 GitHub、VSCode Markdown Preview、Typora 等支持 Mermaid 的工具中直接渲染。

## 1. 端到端主流程

```mermaid
flowchart TD
    A["用户提交诊断请求<br>VIN / 症状 / DTC / 传感器数据"] --> B["FastAPI Router<br>POST /api/vehicle-diagnosis/tasks"]
    B --> C["Pydantic 参数校验<br>VehicleDiagnosisRequest"]
    C --> D["VehicleDiagnosisService.submit_task"]
    D --> E["创建 DiagnosisTask<br>status = submitted"]
    E --> F["Repository 持久化任务<br>MongoDB / SQLite"]
    F --> G["任务入队<br>RedisTaskQueue / InMemoryTaskQueue"]
    G --> H["返回 task_id<br>status = queued"]

    G --> I["Worker 阻塞取任务<br>BLPOP / asyncio.Queue"]
    I --> J["execute_task<br>status = running"]
    J --> K["构建或复用 VehicleDiagnosisGraph<br>Graph Cache + TTL + LRU"]
    K --> L["LangGraph 多 Agent 诊断流程"]
    L --> M["Safety Judge 生成 structured_result"]
    M --> N["保存 state + result<br>status = completed"]
    N --> O["GET /tasks/{task_id}/result"]

    J --> P{执行异常?}
    P -->|超时| Q["status: failed<br>error_message: timeout"]
    P -->|用户取消| R["status: cancelled"]
    P -->|其他异常| S["status: failed<br>保存异常信息"]

    O --> T["维修完成后回填 outcome"]
    T --> U["保存真实根因与维修动作"]
    U --> V["写入 Memory Case<br>vehicle_memory_cases"]
    V --> W["后续相似诊断可检索复用"]
```

## 2. LangGraph Agent 诊断流程

```mermaid
flowchart TD
    START([START]) --> VIN[VIN Context Analyst<br/>车辆档案 / 历史 DTC / 维护记录]

    VIN --> VIN_DECISION{需要工具?}
    VIN_DECISION -->|是| VIN_TOOLS[tools_vin_context]
    VIN_TOOLS --> VIN
    VIN_DECISION -->|否| VIN_CLEAR[Msg Clear VIN Context]

    VIN_CLEAR --> SYM[Symptom Analyst<br/>症状严重度 / 相似案例]
    SYM --> SYM_DECISION{需要工具?}
    SYM_DECISION -->|是| SYM_TOOLS[tools_symptom]
    SYM_TOOLS --> SYM
    SYM_DECISION -->|否| SYM_CLEAR[Msg Clear Symptom]

    SYM_CLEAR --> DTC[Diagnostic Code Analyst<br/>故障码解释 / 组合模式]
    DTC --> DTC_DECISION{需要工具?}
    DTC_DECISION -->|是| DTC_TOOLS[tools_dtc]
    DTC_TOOLS --> DTC
    DTC_DECISION -->|否| DTC_CLEAR[Msg Clear Diagnostic Code]

    DTC_CLEAR --> TEL[Telemetry Analyst<br/>传感器快照 / 时序 / 事件 / 规则]
    TEL --> TEL_DECISION{需要工具?}
    TEL_DECISION -->|是| TEL_TOOLS[tools_telemetry]
    TEL_TOOLS --> TEL
    TEL_DECISION -->|否| TEL_CLEAR[Msg Clear Telemetry]

    TEL_CLEAR --> KNOW[Knowledge Analyst<br/>知识库 / 维修案例]
    KNOW --> KNOW_DECISION{需要工具?}
    KNOW_DECISION -->|是| KNOW_TOOLS[tools_knowledge]
    KNOW_TOOLS --> KNOW
    KNOW_DECISION -->|否| KNOW_CLEAR[Msg Clear Knowledge]

    KNOW_CLEAR --> HR[Hypothesis Researcher<br/>提出故障假设]
    HR --> DEBATE{达到辩论轮数?}
    DEBATE -->|否| CR[Counterfactual Researcher<br/>反证与风险提醒]
    CR --> HR
    DEBATE -->|是| PLAN[Diagnostic Planner<br/>综合证据生成诊断计划]

    PLAN --> ADVISOR[Repair Advisor<br/>维修建议 / 费用区间 / 驾驶建议]
    ADVISOR --> SAFETY[Safety Analyst<br/>安全风险评估]
    SAFETY --> SAFETY_LOOP{达到安全讨论轮数?}
    SAFETY_LOOP -->|否| ADVISOR
    SAFETY_LOOP -->|是| JUDGE[Safety Judge<br/>生成最终结构化结果]
    JUDGE --> END([END])
```

## 3. 工具调用与 Fallback 流程

```mermaid
flowchart TD
    A[分析师节点运行] --> B{LLM 是否可用?}
    B -->|可用| C[尝试 LLM 工具决策<br/>bind_tools / tool_calls]
    C --> D{返回有效 tool_calls?}
    D -->|是| E[进入对应 tools_xxx 节点]
    D -->|否| F[使用默认规则工具策略]
    B -->|不可用| F
    F --> E

    E --> G{工具调用是否超限?}
    G -->|超限| H[跳过超额工具<br/>写入 tool_errors]
    G -->|未超限| I[执行工具<br/>timeout + retries]

    I --> J{工具执行成功?}
    J -->|成功| K[解析工具结果<br/>合并到 state]
    J -->|失败| L[记录错误<br/>tool_errors + ok=false]

    H --> M[回到分析师节点]
    K --> M
    L --> M
    M --> N[基于现有证据生成报告]
    N --> O[写入 analyst_conclusions]
    O --> P[Msg Clear 压缩上下文]
    P --> Q[进入下一个节点]
```

## 4. 数据源 Fallback 流程

```mermaid
flowchart TD
    A[工具请求数据<br/>VIN / Sensor / Event / Repair Case] --> B{VEHICLE_DIAGNOSIS_DATA_PROVIDER}

    B -->|mock| C[跳过 MongoDB<br/>直接使用 mock/local 数据]
    B -->|mongo| D[强制连接 MongoDB]
    B -->|auto| E[优先尝试 MongoDB]

    D --> F{Mongo 可用且查询成功?}
    F -->|是| G[返回 Mongo 数据]
    F -->|否| H[抛出异常<br/>由工具节点记录 tool_errors]

    E --> I{Mongo 可用且有数据?}
    I -->|是| G
    I -->|否| J[Fallback 到 mock/local 数据]

    C --> K[返回可演示数据]
    J --> K
    G --> L[工具结果写入 state]
    K --> L
```

## 5. Memory 读写闭环

```mermaid
flowchart TD
    A[当前诊断任务完成] --> B[用户/维修人员回填 outcome]
    B --> C[提取确认根因<br/>confirmed_root_cause]
    B --> D[提取维修动作<br/>repairs_performed]
    B --> E[提取是否解决<br/>resolved]

    C --> F[构造 recommendation]
    D --> F
    E --> F

    A --> G[构造 situation<br/>VIN / vehicle / symptoms / DTC / telemetry_findings]
    G --> H[VehicleDiagnosisMemory.add_case]
    F --> H
    H --> I{Memory Store 类型}
    I -->|Mongo| J[vehicle_memory_cases 集合]
    I -->|SQLite| K[vehicle_memory_cases 表]
    I -->|InMemory| L[进程内列表]

    J --> M[后续 Diagnostic Planner]
    K --> M
    L --> M

    M --> N[构造当前 situation]
    N --> O[get_memories<br/>token overlap 检索]
    O --> P[返回相似历史案例]
    P --> Q[进入 diagnostic_plan.similar_memory_cases]
    Q --> R[辅助排序故障假设与置信度]
```
